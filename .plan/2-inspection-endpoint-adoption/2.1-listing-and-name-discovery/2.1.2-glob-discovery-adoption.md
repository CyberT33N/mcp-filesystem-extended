### IU-001: Glob discovery traversal adoption
**Type:** `schema`

**Description:**
Adopt the shared traversal policy in the glob discovery handler and schema.

**Ist-Zustand:**
[`getFindFilesByGlobRootResult()`](src/domain/inspection/find-files-by-glob/handler.ts:36) traverses directories recursively and evaluates caller `excludeGlobs` directly; no default vendor exclusion or explicit re-include contract exists.

**Soll-Zustand:**
Glob discovery resolves traversal scope through the shared policy, supports additive re-includes, and preserves explicit root access.

**Architectural Rationale:**
All traversal-based discovery surfaces must share the same default safety model.

**Exact Target:**
[`find-files-by-glob/handler.ts`](src/domain/inspection/find-files-by-glob/handler.ts), [`find-files-by-glob/schema.ts`](src/domain/inspection/find-files-by-glob/schema.ts)

**Implementation Contract:**
Consume the shared traversal policy before recursive descent, keep the existing result structure, and add new optional traversal control fields without renaming `glob` or `excludeGlobs`.

**Scenario-Specific Specification:**
| Field | Value |
|------|-------|
| Target Artifact | Glob handler and schema |
| Full Target Shape / Value | Shared traversal-policy consumption plus additive override fields |
| Authoritative Source | Shared traversal policy from Unit 1 |
| Determination Rule | Broad roots exclude default vendor/cache trees unless explicitly reopened or explicitly rooted |
| Exact Placement | `src/domain/inspection/find-files-by-glob/` |
| Consumers | Glob-based MCP callers and downstream contract docs |
| No-Breaking Constraint | Preserve tool name, `glob` parameter, and structured result shape |

## Actions
[INTENT: ANWEISUNG]
1. **Refactor the glob handler to consume the shared traversal policy**
   - **What:** Replace raw endpoint-local exclude evaluation with shared traversal-policy resolution.
   - **Where:** [`find-files-by-glob/handler.ts`](src/domain/inspection/find-files-by-glob/handler.ts)
   - **How:** Apply the shared policy before recursive descent so vendor/cache trees are skipped by default under broad roots and explicit roots remain valid.
   - **Why:** The glob endpoint must stop behaving differently from the other traversal-based discovery surfaces.
   - **Inputs / References:** [`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts)
   - **Output Contract:** The handler traverses only the effective shared traversal scope.
   - **Verification:** A broad-root glob search no longer walks `node_modules` by default, while an explicit root inside `node_modules` still works.
   - **Stop if:** The implementation would require changing the result shape or the tool name.
2. **Extend the schema additively**
   - **What:** Add new optional traversal override fields to the glob schema.
   - **Where:** [`find-files-by-glob/schema.ts`](src/domain/inspection/find-files-by-glob/schema.ts)
   - **How:** Keep `glob` and `excludeGlobs` intact and add optional `respectGitIgnore` / `includeExcludedGlobs` style fields.
   - **Why:** Agents must be able to explicitly reopen excluded subtrees without losing the hardened default.
   - **Inputs / References:** [`find-files-by-glob/schema.ts`](src/domain/inspection/find-files-by-glob/schema.ts)
   - **Output Contract:** The schema remains backward-compatible and gains explicit opt-in control.
   - **Verification:** Existing callers remain valid and new callers can request optional `.gitignore` participation or explicit re-includes.
   - **Stop if:** The new fields would collide with existing semantics or require a schema reset.
