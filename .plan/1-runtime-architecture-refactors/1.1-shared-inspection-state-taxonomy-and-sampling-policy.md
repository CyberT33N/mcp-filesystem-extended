### IU-001: Shared inspection state contract
**Type:** `object`

**Description:**
Create the reusable shared contract that expresses the inspection content-state taxonomy and the evidence returned by state classification.

**Ist-Zustand:**
The current codebase has only `TextBinaryClassification` with boolean eligibility semantics.

**Soll-Zustand:**
One shared contract expresses richer inspection states and evidence surfaces that later consumers can route on explicitly.

**Architectural Rationale:**
Hybrid files cannot be modeled safely with only `true|false` eligibility.

**Exact Target:**
`src/domain/shared/search/inspection-content-state.ts`

**Implementation Contract:**
Export the exact state taxonomy, confidence/evidence fields, and a concise rationale surface. The contract must be reusable by search, read, and count endpoints.

### IU-002: Shared sampling policy constants
**Type:** `constant`

**Description:**
Move multi-window sample constants into the shared guardrail registry.

**Ist-Zustand:**
The current classifier uses a local first-window sample constant.

**Soll-Zustand:**
Sampling constants are centralized in the shared guardrail registry and expressed as explicit runtime policy values.

**Architectural Rationale:**
Sampling is a cross-endpoint runtime policy concern, not a file-local classifier detail.

### IU-003: Compatibility-preserving classifier bridge
**Type:** `function`

**Description:**
Refactor the current classifier so it emits the richer state model while preserving enough compatibility for still-unmigrated consumers.

**Ist-Zustand:**
Consumers receive only a boolean result and a reason string.

**Soll-Zustand:**
Consumers can begin migrating to the new authoritative state field while old callers continue to function until later tasks finish the migration.

**Architectural Rationale:**
The runtime architecture must move forward without forcing a large breaking refactor in a single step.
