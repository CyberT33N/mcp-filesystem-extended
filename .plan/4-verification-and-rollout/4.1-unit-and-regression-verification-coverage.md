## Actions
[INTENT: ANWEISUNG]
1. **Add shared guardrail unit tests**
   - **What:** Create unit tests for shared traversal policy resolution and traversal-budget assertion helpers.
   - **Where:** `test/unit/domain/shared/guardrails/`
   - **How:** Cover default excluded classes, explicit-root passthrough, additive re-includes, and traversal-budget refusal triggers.
   - **Why:** The core guardrail logic should be validated independently of endpoint wiring.
   - **Inputs / References:** [`vitest.unit.config.ts`](vitest.unit.config.ts), [`test/unit/test-setup.ts`](test/unit/test-setup.ts), [`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts), [`traversal-runtime-budget.ts`](src/domain/shared/guardrails/traversal-runtime-budget.ts)
   - **Output Contract:** Deterministic unit coverage exists for the shared traversal guardrail family.
   - **Verification:** The tests fail when default excluded classes or traversal-budget thresholds are violated incorrectly.
   - **Stop if:** The current unit test harness cannot load the new test paths without an unplanned configuration change.
2. **Add broad-root and explicit-root regression tests**
   - **What:** Create regression tests for default excluded traversal and explicit access to excluded roots.
   - **Where:** `test/regression/inspection/`
   - **How:** Build temporary fixture trees inside the tests and exercise traversal-based handlers against broad roots and explicit excluded roots.
   - **Why:** The rollout task needs executable proof that the timeout-prone broad-root behavior is hardened while explicit vendor access still works.
   - **Inputs / References:** [`vitest.regression.config.ts`](vitest.regression.config.ts), [`test/regression/test-setup.ts`](test/regression/test-setup.ts), traversal-based handlers from Unit 2.
   - **Output Contract:** Regression coverage exists for both the hardened default and the preserved explicit access path.
   - **Verification:** The broad-root case excludes `node_modules`-like trees by default and the explicit-root case still returns results.
   - **Stop if:** The regression harness cannot isolate temporary fixtures without unplanned infrastructure changes.
