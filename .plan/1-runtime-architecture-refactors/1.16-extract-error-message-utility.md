# Task 1.16: Extract Shared extractErrorMessage Utility
[INTENT: ANWEISUNG]

## Goal
[INTENT: KONTEXT]

The pattern `error instanceof Error ? error.message : String(error)` appears identically in at least 11 locations across handler files and the shared preflight guardrail module. This task creates a minimal shared utility function `extractErrorMessage(error: unknown): string` in `src/domain/shared/utils/error-message.ts` and replaces every occurrence of the inline pattern with a call to this function. Additionally, the regex search handler is updated to use the `SEARCH_FILE_CONTENTS_BY_REGEX_TOOL_NAME` constant exported by task 1.15, as that update was deferred from 1.15 to avoid a shared-file conflict.

- No inline `error instanceof Error ? error.message : String(error)` expressions remain in the listed consumer files.
