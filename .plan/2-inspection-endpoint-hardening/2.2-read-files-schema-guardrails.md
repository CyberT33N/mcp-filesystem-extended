### IU-001: Exact Schema Changes for `ReadFilesWithLineNumbersArgsSchema`
**Type:** `schema`

**Description:**
Apply the exact finite request-surface limits to the direct file-read tool schema.

**Ist-Zustand:**
The schema only requires `paths.min(1)` and allows unbounded path counts and unbounded path string length.

**Soll-Zustand:**
The schema MUST change exactly as follows:

| Field | Value |
|------|-------|
| Target Artifact | `ReadFilesWithLineNumbersArgsSchema` |
| Full Target Shape / Value | `paths: z.array(z.string().max(PATH_MAX_CHARS)).min(1).max(MAX_GENERIC_PATHS_PER_REQUEST)` |
| Authoritative Source | This task file + shared limit matrix |
| Determination Rule | Use `PATH_MAX_CHARS` and `MAX_GENERIC_PATHS_PER_REQUEST` from the shared limit module |
| Exact Placement | `src/domain/inspection/read-files-with-line-numbers/schema.ts` |
| Consumers | MCP callers and the downstream read-files handler |
| No-Breaking Constraint | Property name remains `paths`; optionality and semantics remain unchanged |

**Architectural Rationale:**
This adds the correct high anti-abuse ceiling at the earliest possible contract point without collapsing legitimate high-cardinality small-file use cases into an aggressively low cap.

## Actions
[INTENT: ANWEISUNG]
1. **Update the read-files schema with the exact shared caps**
   - **What:** Modify `ReadFilesWithLineNumbersArgsSchema` to use the exact bounded array and string surface from IU-001.

   - **How:** Import `PATH_MAX_CHARS` and `MAX_GENERIC_PATHS_PER_REQUEST` from the shared limit module and apply them directly in the Zod schema.
   - **Why:** This is the earliest enforceable and caller-visible guardrail layer for the direct file-read endpoint.
