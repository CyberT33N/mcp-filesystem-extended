### IU-001: Recursive count traversal adoption
**Type:** `schema`

**Description:**
Refactor recursive line counting to resolve traversal through the shared policy and expose the same additive traversal override fields as the other recursive inspection endpoints.

**Ist-Zustand:**
[`countLinesInDirectory()`](src/domain/inspection/count-lines/handler.ts:320) walks recursive directory trees with endpoint-local exclude checks and no default vendor/cache exclusion policy.

**Soll-Zustand:**
Recursive line counting uses the shared traversal policy, broad-root counts skip default-excluded trees by default, and callers can explicitly reopen excluded subtrees via additive overrides.

**Architectural Rationale:**
Recursive line counts should follow the same default-scope hardening model as discovery and regex traversal.

**Exact Target:**
[`count-lines/handler.ts`](src/domain/inspection/count-lines/handler.ts), [`count-lines/schema.ts`](src/domain/inspection/count-lines/schema.ts)

**Implementation Contract:**
Route recursive traversal through the shared traversal policy, keep the existing line-count result structure, and add new optional override fields without removing current filters.

**Scenario-Specific Specification:**
| Field | Value |
|------|-------|
| Target Artifact | Count-lines handler and schema |
| Full Target Shape / Value | Shared traversal-policy consumption plus additive override fields |
| Authoritative Source | Shared traversal policy from Unit 1 |
| Determination Rule | Broad-root recursive counts skip default-excluded trees unless callers explicitly reopen them |
| Exact Placement | `src/domain/inspection/count-lines/` |
| Consumers | Recursive line-count callers and downstream docs |
| No-Breaking Constraint | Preserve tool name, line-count result shape, and existing file filters |

## Actions
[INTENT: ANWEISUNG]
1. **Refactor recursive count traversal to consume the shared policy**
   - **What:** Replace handler-local traversal admission with shared traversal-policy resolution.
   - **Where:** [`count-lines/handler.ts`](src/domain/inspection/count-lines/handler.ts)
   - **How:** Apply the shared policy before descending into directories so vendor/cache trees are skipped by default under broad roots and explicit excluded roots remain possible.
   - **Why:** Recursive counting must align with the same default safety model as the other traversal-based endpoints.
   - **Inputs / References:** [`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts)
   - **Output Contract:** Recursive traversal follows the shared policy instead of raw local `excludeGlobs` checks.
   - **Verification:** Broad-root recursive counts skip `node_modules` by default and explicit roots inside excluded trees still work.
   - **Stop if:** The implementation would require a new result shape or a tool rename.
2. **Extend the schema additively**
   - **What:** Add the same additive traversal override fields used by the other recursive inspection endpoints.
   - **Where:** [`count-lines/schema.ts`](src/domain/inspection/count-lines/schema.ts)
   - **How:** Preserve `includeGlobs` and `excludeGlobs`, then add optional `respectGitIgnore` / `includeExcludedGlobs` semantics.
   - **Why:** The caller contract must stay consistent for agents moving across traversal-based tools.
   - **Inputs / References:** [`count-lines/schema.ts`](src/domain/inspection/count-lines/schema.ts)
   - **Output Contract:** The count-lines schema remains backward-compatible and gains the new additive controls.
   - **Verification:** Existing callers remain valid and new override fields are explicit.
   - **Stop if:** A schema reset or a rename becomes necessary.
