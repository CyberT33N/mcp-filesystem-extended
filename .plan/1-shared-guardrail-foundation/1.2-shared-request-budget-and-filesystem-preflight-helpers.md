### IU-001: Shared Text Budget Helper Surface
**Type:** `function`

**Description:**
Create the reusable text-budget module that estimates response sizes before content is emitted and throws canonical guardrail failures when projected or actual sizes exceed endpoint-family limits.

**Ist-Zustand:**
No shared text-budget helpers exist; handlers currently serialize unbounded strings directly.

**Soll-Zustand:**
`src/domain/shared/guardrails/text-response-budget.ts` exports these exact functions and constants:

| Field | Value |
|------|-------|
| Function Name | `estimateTokenLoadFromBytes` |
| Signature | `(byteSize: number) => number` |
| Return Type | `number` |
| Inputs | `byteSize` |
| Side Effects | none |
| Call Sites | read-files preflight, regex candidate-byte budgeting, future global response fuse |
| Error Behaviour | none |
| Exact Placement | `src/domain/shared/guardrails/text-response-budget.ts` |
| No-Breaking Constraint | Pure helper; no existing call sites are renamed |

Additional exact helper surface:
- `estimateLineNumberedResponseCharsFromBytes(byteSize: number): number` using `Math.ceil(byteSize * 1.35) + 64`
- `estimateDiffResponseCharsFromByteSizes(leftByteSize: number, rightByteSize: number): number` using `Math.ceil((leftByteSize + rightByteSize) * 1.6) + 512`
- `assertProjectedTextBudget(toolName: string, projectedChars: number, hardCapChars: number, summary: string): void`
- `assertActualTextBudget(toolName: string, actualChars: number, hardCapChars: number, summary: string): void`

**Architectural Rationale:**
The helpers separate deterministic projection from endpoint-specific orchestration and keep all budget refusals aligned to the shared error contract.

**Exact Target:**
`src/domain/shared/guardrails/text-response-budget.ts`

**Implementation Contract:**
The helper module MUST import the shared error-contract builders instead of throwing raw strings, and MUST keep the byte-to-text estimation formulas exactly as specified above.

### IU-002: Shared Filesystem Metadata Preflight Surface
**Type:** `function`

**Description:**
Create the reusable metadata preflight helper used by file-based endpoints before any `fs.readFile` loop begins.

**Ist-Zustand:**
Handlers validate paths individually and read content immediately without a shared metadata-first admission control surface.

**Soll-Zustand:**
`src/domain/shared/guardrails/filesystem-preflight.ts` exports these exact surfaces:

| Field | Value |
|------|-------|
| Function Name | `collectValidatedFilesystemPreflightEntries` |
| Signature | `(requestedPaths: string[], allowedDirectories: string[]) => Promise<Array<{ requestedPath: string; validPath: string; type: "file" | "directory" | "other"; size: number }>>` |
| Return Type | `Promise<Array<...>>` |
| Inputs | requested paths, allowed directory roots |
| Side Effects | reads metadata only |
| Call Sites | read-files handler, regex-search handler, future file-diff preflight |
| Error Behaviour | throws canonical `metadata_preflight_rejected` failure when any path is invalid or missing |
| Exact Placement | `src/domain/shared/guardrails/filesystem-preflight.ts` |
| No-Breaking Constraint | Shared helper only; endpoint response contracts remain controlled elsewhere |

Additional exact helper surface:
- `sumPreflightBytes(entries): number`
- `assertCandidateByteBudget(toolName: string, totalBytes: number, hardCapBytes: number, summary: string): void`
- `assertExpectedFileTypes(toolName: string, entries, allowedTypes: Array<"file" | "directory">): void`

**Architectural Rationale:**
This helper enforces the “metadata first, file content second” principle required by the target architecture and prevents duplicated stat/realpath logic in individual handlers.

**Exact Target:**
`src/domain/shared/guardrails/filesystem-preflight.ts`

**Implementation Contract:**
The helper MUST reuse `validatePath()` and `getFileSystemEntryMetadata()` instead of duplicating path and stat logic.
