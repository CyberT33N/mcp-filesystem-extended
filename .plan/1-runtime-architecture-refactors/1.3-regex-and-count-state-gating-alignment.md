## Information Units
[INTENT: SPEZIFIKATION]

### IU-001: Regex state gating
**Type:** `function`

**Description:**
Refactor regex runtime gating so it explicitly consumes the shared inspection-state taxonomy and stays text-first.

**Ist-Zustand:**
Regex search currently relies on the older boolean classifier semantics.

**Soll-Zustand:**
Regex search explicitly distinguishes text-safe execution from unsupported non-text states.

**Architectural Rationale:**
Regex should not silently broaden into hybrid capability just because the shared taxonomy becomes richer.

### IU-002: Count total-only unsupported-state behavior
**Type:** `function`

**Description:**
Refactor total-only counting so unsupported non-text states are surfaced explicitly.

**Ist-Zustand:**
Total-only counts can still traverse and count text-ish lines without an explicit content-state contract.

**Soll-Zustand:**
The handler explicitly distinguishes supported text surfaces from unsupported or semantically misleading non-text surfaces.

**Architectural Rationale:**
Agents should not interpret unsupported-state zeros as trustworthy content evidence.

### IU-003: Count pattern-aware reroute guidance
**Type:** `object`

**Description:**
Extend the count query policy and handler so unsupported non-text states produce explicit reroute guidance.

**Ist-Zustand:**
The pattern-aware lane reuses the shared native-search lane but does not yet expose the shared inspection-state semantics explicitly.

**Soll-Zustand:**
Unsupported non-text states are surfaced deliberately, with guidance toward fixed-string or byte/cursor inspection where appropriate.

**Architectural Rationale:**
The count endpoint must align with the server-wide inspection contract rather than masking unsupported-state behavior.
