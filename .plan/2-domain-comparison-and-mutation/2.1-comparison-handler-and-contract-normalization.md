## Information Units

### IU-001: File-diff domain normalization
**Type:** `function`

**Description:**
Normalize the file-diff handler-local DTO names and schema export names to direct target-state terminology.

**Ist-Zustand:**
[`handler.ts`](src/domain/comparison/diff-files/handler.ts) still uses generic internal naming such as `FileDiffOperation`, and [`schema.ts`](src/domain/comparison/diff-files/schema.ts) still exports `FileDiffArgsSchema`.

**Soll-Zustand:**
The file-diff domain uses target-state names such as `DiffFilesPair` / equivalent target-state DTO naming and `DiffFilesArgsSchema` for the exported schema constant.

**Architectural Rationale:**
The comparison domain should use the same language as the public tool surface. This reduces ambiguity and avoids application-layer naming translation.

**Exact Target:**
`src/domain/comparison/diff-files/handler.ts`, `src/domain/comparison/diff-files/schema.ts`

**Implementation Contract:**
Rename internal operation DTOs and exported schema identifiers only; preserve the file-pair structure, diff generation flow, and output formatting behavior.

**Scenario-Specific Specification:**
| Field | Value |
|------|-------|
| Function Name | `handleFileDiff` remains stable unless file-local naming requires a target-state alias-free rename within the same domain file |
| Signature | Preserve the existing behavior and argument structure for file-pair arrays |
| Return Type | `Promise<string>` remains unchanged |
| Inputs | File pairs remain left/right path based |
| Side Effects | Reads files and formats diffs only |
| Call Sites | Application registration imports in Unit 3 |
| Error Behaviour | Existing error formatting remains unchanged |
| Exact Placement | `src/domain/comparison/diff-files/handler.ts`, `src/domain/comparison/diff-files/schema.ts` |
| No-Breaking Constraint | No change to file-diff behavior or schema field names |
