# Task 3.1: Comparison Endpoint Guardrails
[INTENT: ANWEISUNG]

## Goal
[INTENT: KONTEXT]
Apply the shared request and response guardrails to the diff endpoints so both file-based and in-memory comparison surfaces have finite request shapes and bounded output sizes. This task keeps the comparison family expressive while preventing pathological caller-supplied raw content from producing oversized diff payloads.

[INTENT: KONTEXT]

- The in-memory text-diff schema and handler reuse the shared raw-content, pair-count, cumulative-input, and family-response caps.

## Context
[INTENT: KONTEXT]
- **Domain:** Comparison-family request/response hardening.
- **Related Files:** `src/domain/comparison/**/*.ts`, shared guardrail modules.
- **Technical Context:** File-based diffs are bounded by actual file sizes, while in-memory text diffs accept caller-supplied raw content and therefore require stricter schema caps.
- **Business Context:** The server must still support useful diff workflows, but it must refuse oversized request payloads and oversized diff outputs predictably.
- **Architectural Context:** Request caps live in the schemas; diff output budgeting lives in the handlers and must reuse the shared helper modules.

## Information Units
[INTENT: SPEZIFIKATION]

### IU-001: Exact Schema Changes for Comparison Endpoints
**Type:** `schema`

**Description:**
Apply the exact request-surface caps to both comparison schemas.

**Ist-Zustand:**
Both comparison schemas allow unbounded pair arrays, and the in-memory diff surface allows unbounded content strings.

**Soll-Zustand:**
Apply these exact schema changes:

| Target Schema | Required Changes |
|------|-------|
| `DiffFilesArgsSchema` | `pairs.max(MAX_COMPARISON_PAIRS_PER_REQUEST)`, each `leftPath.max(PATH_MAX_CHARS)`, each `rightPath.max(PATH_MAX_CHARS)` |
| `DiffTextContentArgsSchema` | `pairs.max(MAX_RAW_TEXT_DIFF_PAIRS_PER_REQUEST)`, each `leftContent.max(RAW_CONTENT_MAX_CHARS)`, each `rightContent.max(RAW_CONTENT_MAX_CHARS)`, each optional label `.max(LABEL_MAX_CHARS)` |

Additionally, the in-memory diff handler MUST later enforce `MAX_TOTAL_RAW_TEXT_REQUEST_CHARS` across all pair content values.

**Architectural Rationale:**
This keeps file-diff request shaping generous but finite, while giving the raw-content diff surface a stricter contract because callers can inject large in-memory text directly.

**Exact Target:**
`src/domain/comparison/diff-files/schema.ts`, `src/domain/comparison/diff-text-content/schema.ts`

**Implementation Contract:**
Import shared constants; do not inline numbers locally.

### IU-002: Exact Runtime Budget Policy for Comparison Handlers
**Type:** `constraint`

**Description:**
Define the exact output and cumulative-input budget behavior for comparison handlers.

**Ist-Zustand:**
Handlers produce full unified diffs without any shared output-budget enforcement.

**Soll-Zustand:**
Use these exact runtime policies:

| Handler | Required Runtime Policy |
|------|-------|
| `handleFileDiff` | After the unified diff is produced, call `assertActualTextBudget("diff_files", actualChars, FILE_DIFF_RESPONSE_CAP_CHARS, "File diff output exceeds the file-diff family cap.")` |
| `handleContentDiff` | Before diff generation, sum all input content lengths across all pairs and refuse if the total exceeds `MAX_TOTAL_RAW_TEXT_REQUEST_CHARS`; after diff generation, enforce `TEXT_DIFF_RESPONSE_CAP_CHARS` on the output |

**Architectural Rationale:**
The handlers must separately protect against oversized request content and oversized diff output without changing successful response semantics.

**Exact Target:**
`src/domain/comparison/diff-files/handler.ts`, `src/domain/comparison/diff-text-content/handler.ts`

**Implementation Contract:**
No truncation is allowed; the handlers must refuse oversize diff outputs.

## Actions
[INTENT: ANWEISUNG]
1. **Update both comparison schemas to use the exact shared caps**
   - **What:** Apply the exact schema constraints from IU-001.
   - **Where:** `src/domain/comparison/diff-files/schema.ts`, `src/domain/comparison/diff-text-content/schema.ts`
   - **How:** Import the shared constants and add `.max(...)` constraints to pair arrays, paths, content strings, and labels.
   - **Why:** The SDK must expose finite request surfaces before handlers run.
   - **Inputs / References:** shared limit matrix, current schema files.
   - **Output Contract:** Both schemas now have explicit finite ceilings and unchanged field names.
   - **Verification:** Confirm no pair array or raw content string remains unbounded.
   - **Stop if:** The schema changes would require renaming existing properties.
2. **Add output budget enforcement to file-based diffs**
   - **What:** Update `handleFileDiff` to enforce the file-diff family output cap.
   - **Where:** `src/domain/comparison/diff-files/handler.ts`
   - **How:** Reuse the shared text-budget helper after diff generation and before success is returned.
   - **Why:** File-based unified diffs can still become too large even when the inputs are path-bounded.
   - **Inputs / References:** shared text-budget helper, shared limits, shared error contract.
   - **Output Contract:** Oversize diff outputs are refused rather than truncated.
   - **Verification:** Confirm the handler now has one explicit shared-budget assertion on the final diff text.
   - **Stop if:** The diff budget enforcement would require changing the successful diff format.
3. **Add cumulative-input and output budget enforcement to in-memory text diffs**
   - **What:** Update `handleContentDiff` to enforce both request and output budgets.
   - **Where:** `src/domain/comparison/diff-text-content/handler.ts`
   - **How:** Sum raw input text lengths across all pairs before diff generation, refuse if the cumulative budget is exceeded, then enforce the final text-diff family cap after diff generation.
   - **Why:** This endpoint is the highest-risk comparison surface because callers can inject arbitrary content directly.
   - **Inputs / References:** shared limits, shared text-budget helper, shared error contract.
   - **Output Contract:** Oversize request content or output diffs are refused deterministically.
   - **Verification:** Confirm the handler enforces `MAX_TOTAL_RAW_TEXT_REQUEST_CHARS` before diff generation and `TEXT_DIFF_RESPONSE_CAP_CHARS` after generation.
   - **Stop if:** The handler would need a new override parameter to remain usable.
