## Information Units

### IU-001: Secondary `.gitignore` enrichment and additive re-include contract
**Type:** `object`

**Description:**
Add the secondary traversal-enrichment layer and caller-facing opt-in controls that allow project-local ignore participation and targeted re-inclusion of excluded subtrees.

**Ist-Zustand:**
The project has a root [`.gitignore`](.gitignore), but no traversal endpoint reads it. Callers only have `excludeGlobs`, and no additive re-include contract exists for mixed broad-root searches.

**Soll-Zustand:**
The shared traversal policy can optionally augment its default excludes with root-local `.gitignore` rules when callers opt in, and callers can selectively re-include excluded subtrees through an additive override parameter such as `includeExcludedGlobs`.

**Architectural Rationale:**
The server needs a stable primary safety policy while still letting callers align with project-local ignore hygiene and reopen specific excluded descendants when the task truly requires it.

**Exact Target:**
[`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts), [`gitignore-traversal-enrichment.ts`](src/domain/shared/guardrails/gitignore-traversal-enrichment.ts), [`package.json`](package.json)

**Implementation Contract:**
Use a dedicated `.gitignore` parser dependency, keep the feature optional, and ensure additive re-includes can reopen excluded descendants without disabling the full default policy globally.

**Scenario-Specific Specification:**
| Field | Value |
|------|-------|
| Target Artifact | [`gitignore-traversal-enrichment.ts`](src/domain/shared/guardrails/gitignore-traversal-enrichment.ts) plus shared policy integration |
| Full Target Shape / Value | Optional root-local `.gitignore` reader plus additive `includeExcludedGlobs` semantics |
| Authoritative Source | [`.gitignore`](.gitignore), [`package.json`](package.json), and the user-approved architecture in [`PLAN.md`](PLAN.md) |
| Determination Rule | `respectGitIgnore` is optional and default-off; `includeExcludedGlobs` reopens only explicitly named excluded descendants |
| Exact Placement | `src/domain/shared/guardrails/` and `package.json` |
| Consumers | All traversal-based inspection endpoints via later tasks |
| No-Breaking Constraint | Existing `excludeGlobs` stays intact and new parameters remain additive |

### IU-002: Traversal runtime budgets and refusal surfaces
**Type:** `object`

**Description:**
Add traversal-entry, traversal-directory, and soft traversal-time budgets that fail deterministically before broad-root traversal can hit environment timeouts.

**Ist-Zustand:**
Regex search has candidate-byte and result-count budgets, but no generic traversal-entry or traversal-directory refusal layer protects broad directory walks before they fan out.

**Soll-Zustand:**
Shared traversal runtime-budget helpers and canonical refusal messaging exist so traversal-heavy endpoints can fail with stable `runtime_budget_exceeded` guidance before environment-level timeouts occur.

**Architectural Rationale:**
Timeout prevention must happen inside the server guardrail model, not via environment timeout tuning alone.

**Exact Target:**
[`traversal-runtime-budget.ts`](src/domain/shared/guardrails/traversal-runtime-budget.ts), [`tool-guardrail-limits.ts`](src/domain/shared/guardrails/tool-guardrail-limits.ts), [`tool-guardrail-error-contract.ts`](src/domain/shared/guardrails/tool-guardrail-error-contract.ts)

**Implementation Contract:**
Add shared counters and assertions for visited entries, visited directories, and optional soft wall-clock budget, reusing the existing runtime-budget refusal family instead of inventing a separate endpoint-local error shape.

**Scenario-Specific Specification:**
| Field | Value |
|------|-------|
| Target Artifact | Shared traversal runtime-budget helpers and existing guardrail constant / refusal modules |
| Full Target Shape / Value | Exported budget constants plus assertion helpers for traversal entries, directories, and soft traversal time |
| Authoritative Source | [`tool-guardrail-limits.ts`](src/domain/shared/guardrails/tool-guardrail-limits.ts) and [`tool-guardrail-error-contract.ts`](src/domain/shared/guardrails/tool-guardrail-error-contract.ts) |
| Determination Rule | Traversal fails early with deterministic refusal text before the environment timeout becomes the primary guardrail |
| Exact Placement | `src/domain/shared/guardrails/` |
| Consumers | Regex search, glob discovery, listing, name discovery, and recursive count traversal |
| No-Breaking Constraint | Reuse the existing runtime budget refusal family and keep public tool names unchanged |

## Actions
[INTENT: ANWEISUNG]
1. **Add secondary `.gitignore` enrichment support**
   - **What:** Introduce [`gitignore-traversal-enrichment.ts`](src/domain/shared/guardrails/gitignore-traversal-enrichment.ts) and add the `ignore` package to [`package.json`](package.json).
   - **Where:** `src/domain/shared/guardrails/` and project package metadata.
   - **How:** Parse the nearest root-local `.gitignore` when callers explicitly enable `respectGitIgnore`; keep the feature disabled by default.
   - **Why:** `.gitignore` should be available as a secondary traversal signal without becoming the primary safety authority.
   - **Inputs / References:** [`.gitignore`](.gitignore), [`package.json`](package.json)
   - **Output Contract:** The shared policy can optionally consume `.gitignore` rules via a dedicated adapter.
   - **Verification:** The feature is optional, deterministic, and separate from the default-exclude SSOT.
   - **Stop if:** Full Git ignore semantics would require undocumented repository traversal beyond the root-local scope defined here.
2. **Add additive re-include semantics to the shared policy**
   - **What:** Extend [`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts) with additive re-include handling.
   - **Where:** [`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts)
   - **How:** Merge default excludes, optional `.gitignore` excludes, caller `excludeGlobs`, and additive `includeExcludedGlobs` into one deterministic effective traversal policy.
   - **Why:** Mixed-root searches need a safe way to reopen specific excluded descendants without turning off the entire default policy.
   - **Inputs / References:** [`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts), [`.gitignore`](.gitignore)
   - **Output Contract:** Explicit re-includes can reopen only the caller-requested excluded descendants.
   - **Verification:** Broad roots stay hardened by default; explicit re-includes reopen only matching descendants.
   - **Stop if:** The solution would require removing or redefining the existing `excludeGlobs` parameter.
3. **Add traversal runtime budgets and deterministic refusals**
   - **What:** Introduce [`traversal-runtime-budget.ts`](src/domain/shared/guardrails/traversal-runtime-budget.ts) and extend guardrail constants / refusal helpers.
   - **Where:** Shared guardrails.
   - **How:** Add visited-entry, visited-directory, and soft traversal-time budgets in [`tool-guardrail-limits.ts`](src/domain/shared/guardrails/tool-guardrail-limits.ts), then reuse [`createRuntimeBudgetExceededFailure()`](src/domain/shared/guardrails/tool-guardrail-error-contract.ts:380) and related formatting helpers.
   - **Why:** The server should refuse oversized traversal scopes deterministically before the environment kills the request.
   - **Inputs / References:** [`tool-guardrail-limits.ts`](src/domain/shared/guardrails/tool-guardrail-limits.ts), [`tool-guardrail-error-contract.ts`](src/domain/shared/guardrails/tool-guardrail-error-contract.ts)
   - **Output Contract:** Shared traversal-budget assertions exist for later endpoint tasks.
   - **Verification:** Budget failures use the canonical runtime-budget refusal family and produce deterministic narrowing guidance.
   - **Stop if:** The only possible implementation would add endpoint-local refusal wording instead of reusing the shared contract.
