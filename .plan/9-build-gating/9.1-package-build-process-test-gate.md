## Information Units

### IU-001: Deterministic build-gate test aggregate
**Type:** `constant`

**Description:**
Create one dedicated aggregate script used only for non-interactive build gating.

**Ist-Zustand:**
[`package.json`](package.json) currently has `test`, `test:unit`, `test:integration`, `test:regression`, and `build`, but no dedicated non-watch build-gate aggregate.

**Soll-Zustand:**
Add `test:build-gate` with the exact command `pnpm test:unit && pnpm test:integration && pnpm test:regression`.

**Architectural Rationale:**
The existing `test` script is watch-oriented (`vitest`) and therefore unsuitable as a deterministic build-precondition surface.

### IU-002: Build lifecycle hook
**Type:** `property`

**Description:**
Attach the aggregate test gate to the package build lifecycle.

**Ist-Zustand:**
The build lifecycle contains only `build: "tsup"` and does not gate build execution on automated tests.

**Soll-Zustand:**
Add `prebuild: "pnpm test:build-gate"` while preserving `build: "tsup"` unchanged.

**Architectural Rationale:**
The lifecycle hook cleanly enforces pre-build test success without altering the underlying build tool binding.
