## Information Units
[INTENT: SPEZIFIKATION]

### IU-001: Shared traversal policy contract
**Type:** `object`

**Description:**
Introduce a single shared traversal policy module that every traversal-based inspection endpoint can consume instead of re-implementing exclusion semantics locally.

**Ist-Zustand:**
Traversal-based endpoints only honor caller-supplied exclusions. Broad roots such as `.` therefore traverse vendor, cache, and virtual-environment trees when the caller does not supply explicit excludes.

**Soll-Zustand:**
A shared module defines canonical default excluded classes, relative-path normalization, and explicit-root passthrough semantics for traversal-based inspection workflows.

**Architectural Rationale:**
The server needs a single traversal-policy truth surface so broad-root searches do not depend on caller luck, endpoint-local heuristics, or project hygiene debt.

**Exact Target:**
[`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts)

**Implementation Contract:**
Expose pure helpers that classify default-excluded directory classes, determine when an explicit root should bypass those defaults, and normalize traversal matching so later endpoint tasks can reuse the same contract.

**Scenario-Specific Specification:**
| Field | Value |
|------|-------|
| Target Artifact | [`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts) |
| Full Target Shape / Value | Shared constants for default excluded classes plus pure traversal-policy resolution helpers |
| Authoritative Source | Current traversal handlers plus the architectural decision recorded in [`PLAN.md`](PLAN.md) |
| Determination Rule | Broad roots exclude high-confidence vendor/cache/build trees by default; explicit roots inside those trees stay allowed |
| Exact Placement | `src/domain/shared/guardrails/traversal-scope-policy.ts` |
| Consumers | Listing, name discovery, glob, regex, count-lines, registration docs, tests |
| No-Breaking Constraint | Keep public tool names and low-level allowed-directory enforcement unchanged |

## Actions
[INTENT: ANWEISUNG]
1. **Create the shared traversal policy module**
   - **What:** Add [`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts) as the canonical shared traversal contract.
   - **Where:** `src/domain/shared/guardrails/`
   - **How:** Define exported constants for default excluded classes and pure helpers for relative-path normalization, excluded-root detection, and effective traversal-policy resolution.
   - **Why:** Every traversal-based endpoint must consume one SSOT instead of carrying its own exclusion logic.
   - **Inputs / References:** [`list-directory-entries/handler.ts`](src/domain/inspection/list-directory-entries/handler.ts), [`find-paths-by-name/helpers.ts`](src/domain/inspection/find-paths-by-name/helpers.ts), [`find-files-by-glob/handler.ts`](src/domain/inspection/find-files-by-glob/handler.ts), [`search-file-contents-by-regex/handler.ts`](src/domain/inspection/search-file-contents-by-regex/handler.ts)
   - **Output Contract:** One exported shared module exists and is ready for downstream endpoint adoption.
   - **Verification:** The module can express default-excluded classes and explicit-root passthrough without importing endpoint-specific code.
   - **Stop if:** The implementation would require moving access checks out of [`validatePath()`](src/infrastructure/filesystem/path-guard.ts:21).
2. **Encode the default excluded class registry**
   - **What:** Define the default excluded category list that broad-root traversal must treat as excluded by default.
   - **Where:** [`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts)
   - **How:** Model high-confidence vendor/cache/build classes such as `node_modules`, `.pnpm`, `.git`, `.venv`, `venv`, `__pypackages__`, `dist`, `build`, and `coverage` as server-owned traversal defaults.
   - **Why:** The default policy must not depend solely on caller-supplied `excludeGlobs` or project-local `.gitignore` quality.
   - **Inputs / References:** [`PLAN.md`](PLAN.md), [`README.md`](README.md)
   - **Output Contract:** The default excluded classes are discoverable from one server-owned location.
   - **Verification:** The registry can be reused by endpoint handlers and tests without duplicating literals.
   - **Stop if:** The implementation would hide the class list in endpoint-local code or registration text.
3. **Preserve explicit-root access semantics**
   - **What:** Encode the rule that explicit roots inside default-excluded trees remain allowed.
   - **Where:** [`traversal-scope-policy.ts`](src/domain/shared/guardrails/traversal-scope-policy.ts)
   - **How:** Distinguish broad-root traversal pruning from explicitly targeted excluded roots so explicit vendor access stays possible without changing [`path-guard.ts`](src/infrastructure/filesystem/path-guard.ts).
   - **Why:** The server must harden broad-root defaults without blocking targeted inspection of vendor declarations or similar explicit paths.
   - **Inputs / References:** [`path-guard.ts`](src/infrastructure/filesystem/path-guard.ts), [`package.json`](package.json)
   - **Output Contract:** Explicit roots and broad roots behave differently by design, not by accident.
   - **Verification:** The shared policy can report that a root under `node_modules` is explicit and therefore allowed for later endpoint tasks.
   - **Stop if:** The only feasible solution would be a low-level path-guard blacklist.
