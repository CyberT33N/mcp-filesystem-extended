# Task 3.3: Registration presets and server-scope tool isolation
[INTENT: ANWEISUNG]

## Goal
[INTENT: KONTEXT]
Extract the shared application-layer registration presets and isolate the remaining server-scope-only tool registration into dedicated application modules.
This task prepares the composition root slim-down by removing inline preset ownership and by separating the `list_allowed_directories` registration from domain tool families.

[INTENT: KONTEXT]

- [`register-server-scope-tools.ts`](src/application/server/register-server-scope-tools.ts) exists and registers only the server-scope tool `list_allowed_directories`.
- No existing application file is modified by this task.

### IU-001: Shared registration preset module
**Type:** `object`

**Description:**
Move the shared application-layer annotation and execution preset objects into one dedicated module.

**Ist-Zustand:**
[`register-tool-catalog.ts`](src/application/server/register-tool-catalog.ts) declares `READ_ONLY_LOCAL_TOOL_ANNOTATIONS`, `ADDITIVE_LOCAL_TOOL_ANNOTATIONS`, `IDEMPOTENT_ADDITIVE_LOCAL_TOOL_ANNOTATIONS`, `DESTRUCTIVE_LOCAL_TOOL_ANNOTATIONS`, and `OPTIONAL_TASK_EXECUTION` inline.

**Soll-Zustand:**
[`tool-registration-presets.ts`](src/application/server/tool-registration-presets.ts) exports those preset constants as the single application-owned preset source.

**Architectural Rationale:**
These presets are not domain-specific and should not be re-declared in each extracted registration module. One composition-layer preset module preserves single-source-of-truth discipline.

**Exact Target:**
`src/application/server/tool-registration-presets.ts`

**Implementation Contract:**
Copy the preset values exactly. Do not rename fields inside the annotation or execution objects.

**Scenario-Specific Specification:**
| Field | Value |
|------|-------|
| Target Artifact | `Tool registration preset constants` |
| Full Target Shape / Value | `READ_ONLY_LOCAL_TOOL_ANNOTATIONS`, `ADDITIVE_LOCAL_TOOL_ANNOTATIONS`, `IDEMPOTENT_ADDITIVE_LOCAL_TOOL_ANNOTATIONS`, `DESTRUCTIVE_LOCAL_TOOL_ANNOTATIONS`, `OPTIONAL_TASK_EXECUTION` |
| Authoritative Source | Inline preset constants in [`register-tool-catalog.ts`](src/application/server/register-tool-catalog.ts) |
| Determination Rule | Preserve the exact object values and field names |
| Exact Placement | `src/application/server/tool-registration-presets.ts` |
| Consumers | Extracted application registration modules and final composition root |
| No-Breaking Constraint | Preset values remain unchanged |

### IU-002: Server-scope tool isolation module
**Type:** `function`

**Description:**
Move the server-scope `list_allowed_directories` registration into a dedicated application module.

**Ist-Zustand:**
[`register-tool-catalog.ts`](src/application/server/register-tool-catalog.ts) registers `list_allowed_directories` inline beside domain tool families.

**Soll-Zustand:**
[`register-server-scope-tools.ts`](src/application/server/register-server-scope-tools.ts) exports a registration function that registers only `list_allowed_directories`.

**Architectural Rationale:**
`list_allowed_directories` exposes server-access scope, not domain behavior. It belongs in application/server composition but should still be isolated from the domain-family registration modules.

**Exact Target:**
`src/application/server/register-server-scope-tools.ts`

**Implementation Contract:**
Register exactly one tool named `list_allowed_directories` with the same title, description, annotations, and text response behavior as the monolithic catalog.

**Scenario-Specific Specification:**
| Field | Value |
|------|-------|
| Function Name | `registerServerScopeTools` |
| Signature | Accept the same registration context shape used by the monolithic catalog |
| Return Type | `void` |
| Inputs | Application registration context including `server`, `allowedDirectories`, and `executeTool` if needed |
| Side Effects | Registers one server-scope MCP tool |
| Call Sites | Final composition root in task `3.4` |
| Error Behaviour | Preserve current registration behavior; no new error handling branch |
| Exact Placement | `src/application/server/register-server-scope-tools.ts` |
| No-Breaking Constraint | The tool name, description, and response text remain unchanged |
