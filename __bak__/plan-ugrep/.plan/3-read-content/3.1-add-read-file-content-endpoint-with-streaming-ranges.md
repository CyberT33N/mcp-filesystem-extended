# Task 3.1: Add the `read_file_content` endpoint with streaming ranges and cursor modes
[INTENT: ANWEISUNG]

## Goal
[INTENT: KONTEXT]

Add the new public content-read endpoint that supports bounded full reads for small files and explicit line-, byte-, and cursor-based reads for large files.

This task creates the single-file content-access surface that complements, rather than replaces, the existing bounded multi-file endpoint [`read_files_with_line_numbers`](src/application/server/register-inspection-tool-catalog.ts:83).

## Expected Result
[INTENT: KONTEXT]

- A new public endpoint named `read_file_content` exists.
- The endpoint exposes `full`, `line_range`, `byte_range`, and `chunk_cursor` request modes.
- Large-file content reading uses a dedicated streaming reader instead of `fs.readFile()`.

## Information Units
[INTENT: SPEZIFIKATION]

### IU-001: New public content-read endpoint
**Type:** `constraint`

**Description:**
Add the public endpoint `read_file_content` as the single-file content-access surface for bounded full reads and large-file-safe ranges/cursors.

**Ist-Zustand:**
The current public read surface is the bounded multi-file endpoint `read_files_with_line_numbers`.

**Soll-Zustand:**
The new single-file content endpoint exists while the old bounded batch-read endpoint remains intact.

**Architectural Rationale:**
The platform needs a separate content-access contract for large-file-safe ranged reading.

### IU-002: Streaming reader
**Type:** `function`

**Description:**
Create the dedicated streaming reader for line-range, byte-range, and cursor access.

**Ist-Zustand:**
The current read code uses `fs.readFile()` and whole-file string formatting.

**Soll-Zustand:**
Large-file content access flows through a dedicated streaming reader.

**Architectural Rationale:**
Large content reads must not depend on whole-file in-process loading.

### IU-003: Explicit mode thresholds
**Type:** `constant`

**Description:**
Bind the explicit thresholds for full reads, line ranges, and byte ranges.

**Ist-Zustand:**
No such public endpoint exists.

**Soll-Zustand:**
The new endpoint carries explicit and predictable thresholds that are visible in code and docs.

**Architectural Rationale:**
The caller must know which operation mode is allowed without the endpoint degenerating into an unbounded read surface.
